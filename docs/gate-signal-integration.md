# 全高闸与车辆道闸信号接入

## 车辆道闸

- 数据接口：`GET /parking-api/public/events`
- 设备范围：7、8、9、10、11号门进出通道，共10台设备
- 事件字段：通道、进出方向、识别时间和唯一事件编号
- 播放策略：首次请求建立基线；出现新事件时播放开闸动画，2.5秒后复位

7、8、10号门使用OEM IPNC HTTP适配，9、11号门使用Signalway XML适配。
12号门不接入。车辆数据由私有FastAPI后端统一标准化，前端不直连设备。

## 人员全高闸

- 数据接口：`GET /gateway/personnel/ControllerMonitor/GetNowInfo`
- 事件来源：智能识别、刷卡开门、按钮开门和远程开门
- 播放策略：按设备序列号去重，新事件触发后2.5秒自动复位

现场方向映射如下：

- `192.168.51.100` -> `fullheight_X02`
- `192.168.52.100` -> `fullheight_X03`
- `192.168.53.100` -> `fullheight_X04`
- `192.168.53.118` -> `fullheight_X01`

人员与车辆接口并行采集，单个子系统异常不会阻塞另一个子系统。

## 火车道闸

- 数据接口：`GET /train-barrier-api/stats`
- `192.168.51.103` -> `trainBarrier_01`
- `192.168.51.105` -> `trainBarrier_02`
- 火车到位信号：`192.168.51.105:DI2`
- 两侧关到位反馈：各设备 `DI1`

开杆阶段两侧动画同步打开；关杆阶段按两侧 `DI1` 反馈分别复位。接口中的火车到位
信号和控制阶段同时更新大屏“火车道状态”的“空闲/占用”显示。前端只读取状态，
不通过该接口发送开杆或关杆命令。
